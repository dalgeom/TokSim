### 2. 통계 분석기 ([src/lib/analyzer/statistics.ts](../src/lib/analyzer/statistics.ts))
프론트엔드에서 AI 호출 없이 돌아가는 순수 JS 분석기:
- **참여자별 메시지 수 & 비율**
- **평균 답장 시간**: 다른 사람이 직전에 말한 뒤 6시간 이내 응답한 경우만 집계 (오래된 gap 제외)
- **대화 시작 횟수**: 이전 메시지와 60분 이상 공백이 있으면 "새 대화 시작"으로 카운트
- **ㅋ/ㅎ/ㅠ·ㅜ 문자 총 카운트** + 메시지당 평균 ㅋ 개수
- **글자 수 & 메시지당 평균 글자 수** (text 타입만 집계)
- **미디어 카운트**: 사진/동영상/이모티콘/음성/파일 참여자별 + 전체 총계
- **시간대별 분포** (0~23시 버킷)
- **요일별 분포** (일~토)
- **많이 쓴 단어 TOP 10**: 정규식으로 문장부호 스트립 → 공백 분리 → 길이 2 이상 + 한국어 조사·감탄사 불용어 제거 + ㅋㅎㅠ·숫자 필터링
- **총 대화 일수 & 하루 평균**: **달력 날짜 자정 기준**으로 계산 (후술 버그 수정 반영)

- 상단 요약 카드 3개: 총 메시지 / 대화한 날 / 하루 평균
- 참여자 섹션: 이름·카운트·비율 바 + 평균 답장, 메시지당 글자, ㅋ/ㅎ/ㅠ, 대화 시작, 사진, 이모티콘을 `dl` 그리드로 표시
- 시간대별 세로 바 차트 (24칸)
- 요일별 가로 바 차트
- 많이 쓴 단어 TOP 10 리스트
- 미디어 메시지 카드 그리드
- 480px 이하 모바일 반응형 (카드·p-stats가 1열로 재배치)
- 데이터 없을 때 에러 박스 + 홈으로 돌아가기 버튼

### 4. 랜딩 → 결과 라우팅 ([src/routes/+page.svelte](../src/routes/+page.svelte))
- "분석하기" 클릭 → 파서 호출 → 성공 시 `sessionStorage.setItem('toksim:chatData', ...)` → `goto('/result')`
- 실패 시 빨간 에러 박스 표시 + isProcessing 상태로 버튼 중복 클릭 방지
- 사용법 안내 섹션 추가 (복사 → 붙여넣기 → 분석하기 / 서버 미저장 프라이버시 강조)

**선택 이유**: 프라이버시 원칙상 서버 왕복 없이 브라우저 내에서만 데이터가 흐르는 게 맞음. `sessionStorage`를 택해서 새로고침에도 살아남게 하되, 탭이 닫히면 자동 폐기되도록 함.

## 발견 + 수정한 버그

### 🐛 "대화한 날 = 2" (같은 날 대화인데 2일로 카운트)
**원인**: `totalDays` 계산식이 `Math.ceil((endDate - startDate) / dayMs) + 1` 로 되어있어서, 같은 날 내에서 시작(15:25)과 끝(18:08)이 달라도 `Math.ceil(0.113) + 1 = 2`가 되는 off-by-one 버그.

**수정**: 타임스탬프가 아니라 **달력 날짜 자정 기준**으로 `startMidnight`, `endMidnight`을 만들고 `Math.round(diff / dayMs) + 1`. 같은 날이면 diff = 0 → 1일로 정확하게 계산됨.

## 미처리로 남은 것 / 3일차 이후 과제

### 한국어 토큰화 개선
- **조사 분리 안 됨**: `상의`, `상의만`, `상의로`가 각각 다른 토큰으로 세어짐. 간단한 조사 스트립 룰(`은/는/이/가/을/를/의/로/에/에서/만/도` 끝부분 제거) 추가 고려.
- **유니코드 이모지 필터링**: 😀🎉 같은 이모지가 텍스트에 섞이면 현재 단어 빈도에 들어갈 가능성. 토큰화 단계에서 `\p{Emoji}` 필터 추가 필요.

### 파서 보강
- **텍스트 + 이모티콘 혼합 메시지**: 현재는 exact 매치만 `emoticon`으로 분류. `"ㅋㅋ 이모티콘 귀엽다"` 같은 줄은 `text`로 처리됨. 본문 내 포함 여부까지 감지할지 결정 필요.
- **단체 채팅방**: 3인 이상 참여자가 있을 때 UI가 2명 기준으로만 디자인되어 있음. 참여자 많을 때 스크롤/요약 처리 필요.
